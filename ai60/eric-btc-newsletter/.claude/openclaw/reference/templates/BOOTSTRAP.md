# BOOTSTRAP.md Template

This page provides guidance for an AI agent beginning its operational lifecycle without established memory files. The documentation establishes a conversational framework rather than a formal interrogation approach.

## Key Setup Steps

The template suggests four foundational elements to establish:

1. **Identity components** -- determining a name, character type, communication style, and symbolic representation
2. **User profiling** -- gathering their preferred address, timezone, and relevant context
3. **Values alignment** -- documenting what matters to them and behavioral expectations via a "SOUL.md" file
4. **Connection preferences** -- offering multiple communication channels (web, WhatsApp, Telegram)

## Core Philosophy

The guidance emphasizes organic discovery: *"Don't interrogate. Don't be robotic. Just... talk."* Rather than rigid data collection, the approach prioritizes natural conversation to mutually understand the relationship parameters.

## Documentation Structure

Three memory files capture the outcomes:
- `IDENTITY.md` -- agent characteristics
- `USER.md` -- user information
- `SOUL.md` -- shared values and boundaries

The template concludes by suggesting deletion once initialization completes, signaling transition from setup mode to operational status.
